
================================================================================
                    AGENT ROLES AND RESPONSIBILITIES
================================================================================

1. OrchestratorAgent
------------
Primary role:
Central coordinator of the entire request pipeline.

Responsibilities:
* Receives the raw customer request
* Calls extract_request_items to trigger the RequestExtractionWorker
* Coordinates the overall processing flow
* Sends parsed items for quote construction
* Triggers stock, pricing, and fulfillment-related operations
* Applies quote-level business rules such as discounts
* Calls the final response rendering step
* Handles top-level errors and returns safe fallback messages

Input:
- Raw customer request text
- Agent date

Output:
- Final formatted customer response


2. InventoryAgent
------------
Primary role:
Manages stock-related checks and replenishment operations.

Responsibilities:
* Checks stock for a specific item
* Retrieves inventory status across items
* Calculates supplier delivery dates
* Creates stock replenishment orders
* Validates that the catalog item exists before ordering
* Verifies enough cash is available before stock purchase
* Prevents invalid restocking actions

Input:
- Item name
- Agent date
- Quantity

Output:
- Stock availability details
- Inventory snapshot
- Estimated delivery date
- Stock order transaction details


3. QuotingAgent
------------
Primary role:
Handles quote support through pricing, catalog lookup, and historical references.

Responsibilities:
* Retrieves unit prices for catalog items
* Supports catalog item detail lookup
* Checks whether requested items are available in catalog
* Searches similar historical quotes
* Supplies pricing context for quote construction
* Helps map customer wording to known product records

Input:
- Item name
- Agent date
- Search terms

Output:
- Unit price
- Catalog item details
- Historical quote matches


4. FulfillmentAgent
------------
Primary role:
Processes validated sales and records sale transactions.

Responsibilities:
* Validates catalog existence before sale
* Confirms sufficient stock before transaction
* Rejects invalid sale quantities
* Calculates sale total price
* Writes sale transactions to the database
* Supports partial fulfillment based on available stock
* Returns transaction confirmation details

Input:
- Item name
- Quantity
- Agent date

Output:
- Sale transaction record
- Transaction ID
- Total sale amount


5. RequestExtractionWorker
------------
Primary role:
Parses free-text customer requests into structured JSON order items.

Responsibilities:
* Receives raw customer request text
* Uses the LLM to identify requested products and quantities
* Returns structured JSON only
* Assigns default quantity = 1 when quantity is omitted
* Extracts explicitly mentioned items only
* Does not perform pricing, stock checks, or database writes
* Is instantiated dynamically inside extract_request_items()

Input:
- Raw customer request text

Output:
- Structured JSON payload containing requested items and quantities
